# Collaboration Modes

## Dialogue cell

Discovery Lead is the only user-facing role. A Loop Analyst or Feature Critic may create a bounded read-only note when a loop is ambiguous or a feature could hijack it. They never ask the user questions or write the Brief.

## Build modes

| Mode | Use when | Roles |
| --- | --- | --- |
| `prototype` | One loop, small scope, one uncertain interaction. | Lead, optional Prototype Architect, QA, Reflection. |
| `agent-mvp` | Confirmed in-game Agent changes a core choice or feedback. | Prototype roles plus Product-Agent Designer. |
| `standard` | Independent design/content and art packages are needed. | Lead, design, art, QA, Reflection. |
| `studio` | Sustained content, complex platform, networking, audio, or release scope. | Add only affected technical, audio, or release roles. |
| `takeover` | Existing project lacks ownership or evidence. | Lead, QA, Reflection before changes. |

Lead alone writes integrated source/configuration/Candidate. Contributors deliver named inbound artifacts. Long-lived tasks require user authorization; otherwise create packets only.
