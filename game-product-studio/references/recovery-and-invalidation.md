# Recovery and Invalidation

Any change to confirmed Brief, engine decision, source, configuration, asset, scene, schema, build, or hosting that can alter behavior invalidates affected downstream evidence. Record the change, refresh the collaboration plan if necessary, freeze a new Candidate, and re-run only checks that depend on the changed artifact.

Recover from task card → progress recap → execution log → decision log → confirmed Vibe/Product/Agent briefs → Candidate manifest → QA/Reflection evidence.
