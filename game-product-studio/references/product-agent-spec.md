# Product-Agent Specification Rules

Create a product Agent specification only after the feature has been placed in a confirmed loop. Define player value, trigger, available game-state inputs, allowed outputs, prohibited behavior, player override, silence/frequency control, fallback, and test evidence.

Do not assume an LLM, network, backend, or credential. A deterministic advisor may satisfy the product contract. If a model service is needed, stop for separate user authorization before provider, key, server, dependency, or data handling decisions.
